The Splunk Attack Analyzer SOAR app can be used to connect with the Splunk Attack Analyzer analysis platform

Manual polling:

  • Maximum containers count parameter is used to limit the number of containers to be ingested while polling jobs
  • While performing manual polling, no check is provided to restrict duplicate data(state file is not updated during manual polling)
  • Pagination is implemented to get all the jobs for provided time window.
Schedule polling:
  • In the first cycle, all the data from the entered hour in "since" gets ingested and after the ingestion is the completed, state file is updated with the "UpdatedAt" time of the latest/first record.
  • In the following cycles, all the records from the given hour in "since" gets ingested, and if while ingesting, the app finds "UpdatedAt" time of already ingested record, the ingestion stops

The following actions are supported by the app:

  • Submitting a URL for analysis
  • Submitting a file for analysis
  • Fetching analysis (job) summary data
  • Fetching the forensics for a job
  • Downloading screenshots for a job and attaching them to the vault
  • Downloading an offline PDF report for a job and attaching it to the vault
